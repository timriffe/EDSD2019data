2 Reading a fixed width file

Give read_fwf() a whirl. This worked on the first try after glancing at the options in the help file (type ?read_fwf in the console).

pos <- fwf_positions(
         start = c(9, 13, 19, 21, 23, 75, 147), 
         end = c(12, 14, 20, 22, 23, 76, 148), 
         col_names = c("year", "month", "hour","minute","weekday","mage","fage"))

NAT <- read_fwf(
  here::here("Data","Nat2018PublicUS.c20190509.r20190717.txt"), 
  col_positions = pos)

The pre-step fwf_positions() is a helper function to tell read_fwf() where the columns are that we want to extract. This is the quickest and lightest option, since we dont have to read in the whole dataset, which would max out memory on machines under 4Gb of memory. I got the column positions (start and end vectors) from the pdf documentation that one can download alongside the data. If the providers were a bit more polite they would provide the column positions in a spreadsheet, but this wasnt so arduous. By reading in just columns were interested in the data object is kept lighter and easier to work with. Data like this really shouldnt cause hangups when using these tools. There are other tricks one can use for truly large data, but weve not got into those.

First lets fix the month, hour, and minute variables, which parsed as character due to leading zeros. We can use group_by() and summarize() to tabulate as we please.

         hour = as.integer(hour),

3 Redistribute missing time of day

Check missing time: 316 cases

NAT %>% 
  filter(minute == 99,
         hour == 99) %>% 
  nrow()
## [1] 316

For the exercises I have in mind its probably fine to throw these cases out if using the time variable as its such a small fraction of births. But we could also redistribute them. In this case how? There are people who dedicate themselves to this question. If were lazy we might do so proportional to the distribution of births matched to all the characteristics they have in common. That means we could create a big multidimensional probability distribution of time of day for the combination of knowns for each birth with unknown time of day. Its laborious, maybe also semirigorous. Even more rigorous would be to include information on other variables you might not be interested in for the redistribution. But the more things you include in the redistribution the smaller the cells get, and it ends up being a bunch of noise and 0s. In that case, you might want to smooth the distribution used to redistribute unknowns, and doing so already puts things in a more statistical setting, where its best to take advice from an actual statistician. For problems of this kind, and large count data like these, its common to use one or two variables for such redistributions. Dudel and Klsener (2018) does something similar for missing ages of fathers (rescale by the distribution of known fathers age within each mother age). IMO this is better than imputing means. Before such an exercise wed want to do a tabulation, which will make these data quite a bit lighter.

NAT <- NAT %>% 
  # tabulate by mother age, hour, minute
  group_by(mage, hour, minute) %>% 
  summarize(n = n()) %>% 
  ungroup() %>% 
  # create a "fraction of the day" variable
  mutate(dfrac = ifelse(minute == 99,              # condition
                        NA,                        # true
                        (hour * 60 + minute) / (24*60))) %>% # false
  # changed to retain hour and minute
  select(mage, n, dfrac, hour, minute) %>% 
  # whatever we group by is what we redistribute within
  group_by(mage) %>% 
  # what is the fraction of births in each time-of-day within each mother age?
  mutate(bfrac = n / sum(n[!is.na(dfrac)]),
         n = n + bfrac * # multiply into birth of unknown time
                   ifelse(
                     sum(is.na(dfrac)) == 0, # condition
                     0,                      # true
                     n[is.na(dfrac)])) %>%   # false
  ungroup() %>% 
  # now remove the NA times
  filter(!is.na(dfrac))# %>% 
  # no need to recreate, could have retained them in the first place.
  # and get minute / hour back from the day fraction
  # mutate(minute = floor((dfrac * (60 * 24)) %% 60),
  #        hour = floor((dfrac * (60 * 24)) - minute))

4 Visualize it: surface

Now weve redistributed births of unknown time of day according to the distribution of known time of day within each mother age. The within part of this was dealt with by the group_by(mage) statement, where one could add another variable or two to make it finer. However, already for mother age under ca 15 or over ca 35, were not filling each minute of the day with observations. Were we to zoom in we might even observeminute heaping

library(lubridate)
library(colorspace)
logit <- function(x){
  log(x/(1-x))
}
NAT %>% 
  mutate(time = paste(hour, minute, sep = ":"),
         time = as.POSIXct(time, format = "%H:%M")) %>% 
  filter(mage >= 17,
         mage <= 35) %>% 
  ggplot(mapping = aes(x = time, y = mage, fill = logit(bfrac))) +
  geom_tile() + 
  scale_fill_continuous_sequential("Viridis") +
  scale_x_datetime(date_breaks="2 hour", date_labels="%H:%M", expand = c(0,0))

Youll note a two-step mutate() call that defines the time variable out of hours and minutes. In order to get ggplot() to render x axis as properly formatted hours, its a good idea to convert our integer representation of hours and minutes into a real time class. Time classes are varied and tricky. From ggplot() warnings I learned the POSIXct format was needed, and a this StackOverflow answer gave me what I needed to fix it https://learnr.wordpress.com/2010/02/25/ggplot2-plotting-dates-hours-and-minutes/, so now the x axis looks good in the plot.

5 Visualize it: ridgeplot

We trucate mage in order to remove noise that might distort the pattern, but we could recover more of the surface if it were smoothed. We might be able to recover more definition in the time of day pattern if it were represented as lines. Lets see:

library(ggridges)
NAT %>% 
  # first group to 10 min intervals, minute precision is the source
  # of much of our noise problem
  mutate(min10 = minute - minute %% 10) %>% 
  group_by(mage, hour, min10) %>% 
  summarize(bfrac = sum(bfrac)) %>% 
  ungroup() %>% 
  # now declare hourmin time variable
  mutate(time = paste(hour, min10, sep = ":"),
         time = as.POSIXct(time, format = "%H:%M")) %>% 
  # select age range of interest
  filter(mage %% 5 == 0,
         mage >= 20,
         mage < 50) %>% 
  # ridgeplot following form of previous lesson
  ggplot(mapping = aes(x = time, 
                       y = factor(mage, 
                                  levels = sort(unique(mage)), 
                                  ordered = TRUE))) +
    geom_ridgeline(mapping = aes(x = time, 
                                               height = bfrac), 
                   scale = 70,
                         alpha = .6) +
  scale_x_datetime(date_breaks="2 hour", date_labels="%H:%M", expand = c(0,0))

OK, we could sense some of this variation in the surface plot, but here its a but clearer I think: the 8:00 peak gets sharper as mother age increases, and the 13:00 also emerges with age.

If you dont know what you want to do for your assignment, you might ask: how much of this pattern is due to cesarians? For that, just repeat the same thing twice and plot a double-ridge as we did for ASFR: if they dont overlap itll be obvious. Does this pattern have month seasonality? It is identical for boy and girl births? This is very rich data and youre sure to find something interesting in it.
